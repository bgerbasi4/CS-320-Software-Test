# CS-320-Software-Test
In this course, I focused on a lot on making sure my code was both functional and secure. One of the main ways I did this was by using unit testing with JUnit. By writing tests for my classes like Contact and ContactService, I was able to check that each method worked correctly and also make sure invalid data was handled properly. This helped me catch errors early instead of waiting until the end. I alsofollowed the project requirements closely, especially with input validation, which is imporatant for keeping programs secure and preventing bad data from causing issues.
When it comes to interpreting user needs, I learned that it really starts with understanding the requirements and translating them into specific rules in the code. For example, if a user needs a contact system that stores names and phone numbers, I have to think about limits, like how long a name can be or making sure fields aren't left empty. I used those requirements to build my classes and then wrote tests to confirm that everything worked the way the user would expect. This process helped me see how important it is to think from the user's perpective while coding. 
My approach to designing software has improved a lot during this course. I started by breaking problems down into smaller parts, like creating separate classes for data (Contact) and logic (ContactService). This made the program easier to understand and manage. I also planned out what each method should do before writing code, which saved time later. As I worked through the projects, I got more comfortable using a structured approach where I design first, then implement, and finally test.
Overall, this course helped me understand how testing is not just something you do at the end, but something that should be part of the entire development process. I also feel more confident in writing cleaner, more reliable code and making sure it meets both user needs and technical requirements. These are skills I know I will keep using in furture classes and in my career.
